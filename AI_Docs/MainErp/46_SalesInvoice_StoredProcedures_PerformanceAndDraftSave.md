## Stored Procedures

### `dbo.MainErp_SalesInvoice_Search`

Purpose:

- Fast paged list/search for workshop and pump invoices.

Parameters:

- `@TypeInvoice`
- `@SearchText`
- `@FromDate`
- `@ToDate`
- `@BranchId`
- `@Page`
- `@PageSize`

Filters:

- `Transactions.Transaction_Type IN (21,42,38,9)`
- `TypeInvoice = 2` for pump sales.
- `TypeInvoice <> 2` for workshop sales.

### `dbo.MainErp_SalesInvoice_GetDetails`

Purpose:

- Load the complete read-only invoice workspace in one round trip.

Result sets:

1. Transaction header.
2. `Transaction_Details` lines.
3. `TblSalesPayment` rows.
4. `DOUBLE_ENTREY_VOUCHERS` rows.
5. Related inventory transactions generated by `CreateIssueVoucher2` / `CreateRecieveVoucher`.

### `dbo.MainErp_SalesInvoice_SaveDraft`

Purpose:

- Controlled draft-save gate only.

Safety defaults:

- `@DryRun = 1`
- `@EnableDraftWrite = 0`

When defaults are used, it returns the payload and safety message without writing anything.

If explicitly called with:

- `@DryRun = 0`
- `@EnableDraftWrite = 1`

then it can save only a draft `Transactions` header. It does not save invoice lines and does not create:

- `Notes`
- `DOUBLE_ENTREY_VOUCHERS`
- inventory issue vouchers
- inventory receive vouchers
- QR/e-invoice status
- posting/approval

This procedure is not exposed as an active UI save button yet.

### `dbo.MainErp_SalesInvoice_SavePumpDeferredDistribution`

Purpose:

- controlled dry-run/save gate for the pump deferred customer distribution from `Transaction_Details.DetailsPump`.

Safety defaults:

- `@DryRun = 1`
- `@EnableDraftWrite = 0`

If explicitly enabled later, it can update only:

- `Transaction_Details.DetailsPump`
- `Transaction_Details.Deferred`
- `Transaction_Details.DeferredQty`

It refuses to update closed, posted, or approved invoices.

It does not create accounting or inventory side effects.

## Web Binding

`SalesInvoiceReadRepository` now attempts to use:

- `dbo.MainErp_SalesInvoice_Search`
- `dbo.MainErp_SalesInvoice_GetDetails`

If a database does not have the stored procedures yet, the repository falls back to the previous inline read SQL. This protects legacy testing databases while allowing the high-performance path on upgraded databases.
